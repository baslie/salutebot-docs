# Определяем уровни доступа

Обновлено 19 сентября 2024

Studio](https://developers.sber.ru/docs/ru/studio/overview)[![](/assets/bot-steps/project-creation/roles/salutebot-new.png)
SaluteBot](../../overview.md)

Пользователи могут работать в одном пространстве со следующими ролями:

* Владелец пространства.
* Администратор.
* Менеджер.
* Разработчик.
* Тестировщик.

Роли назначаются при добавлении пользователей в пространство. Каждая роль имеет свои уровни доступа.

## Владелец пространства

Владелец пространства в Studio — это пользователь, в чьем аккаунте было создано пространство. Владелец имеет максимальный уровень доступов в своем пространстве.

Такая роль в пространстве только одна, т. е. в одном пространстве не может быть двух пользователей с ролью «Владелец».

Если владелец пространства потерял доступ к аккаунту или, например, увольняется из компании, есть возможность передать пространство другому человеку. Для этого нужно:

1. Добавить в пространство нового владельца с любой ролью.
2. Новый владелец пространства должен заполнить и отправить форму запроса.
3. Текущий владелец пространства должен отправить запрос на смену владельца на [developer@sberdevices.ru](mailto:developer@sberdevices.ru). В запросе нужно указать:
   1. Данные текущего владельца: ФИО, а также номер телефона или адрес электронной почты, которые указаны в профиле Studio.
   2. Данные нового владельца: ФИО, а также номер телефона или адрес электронной почты, которые указаны в профиле Studio.
   3. Ссылку на пространство, в котором нужно сменить владельца.
   4. Причину передачи роли.

## Доступ к управлению командой

| **Доступы** | **Владелец** | **Администратор** | **Менеджер** | **Разработчик** | **Тестировщик** |
| --- | --- | --- | --- | --- | --- |
| Просмотр состава команды | Да | Да | Нет | Нет | Нет |
| Приглашение других пользователей | Да | Да | Нет | Нет | Нет |
| Редактирование роли других пользователя | Да | Да | Нет | Нет | Нет |
| Удаление пользователя из команды | Да | Да | Нет | Нет | Нет |
