Блок **Готовые интеграции** позволяет настроить доступ к списку интеграций со сторонними системами вашего бизнеса, например Битрикс24 или amoCRM.

Готовые сценарии работают только в [проектах SaluteBot](../../bot-steps/project-creation/overview.md).
Список доступных готовых сценариев зависит от подключенного тарифа.

## Список готовых сценариев

### Битрикс24 Статус заказа

Сценарий интеграции определяет в запросе пользователя желание узнать информацию о заказе и передает необходимые данные в модуль «Интернет-магазин (sale)» системы Битрикс24. Для этого в запросе пользователя определяется номер заказа, если номер отсутствует, может быть задан уточняющий вопрос. Ответ системы Битрикс преобразуется и передается пользователю.

При выборе сценария отображается блок **[Битрикс24] Статус заказа**, который содержит следующие поля:

* Поле **Bitrix Webhook**. В поле необходимо указать вебхук, [полученный в системе Битрикс24](https://dev.1c-bitrix.ru/learning/course/?COURSE_ID=99&LESSON_ID=8581) .

  При создании вебхука, в разделе «Настройка прав» нужно выбрать модуль «Интернет-магазин (sale)».
* Поле **Ответ по статусу заказа**. Произвольное сообщение для запроса номера заказа у пользователя.

  Значение по умолчанию: «Пожалуйста, скажите номер вашего заказа».
* Поле **Запрос номера заказа**. Произвольное сообщение для описания статуса заказа.

  Значение по умолчанию: «Ваш заказ номер `{номер заказа}` от `{дата заказа}` находится в статусе "`{статус заказа}`"».
* Поле **Ответ, если заказ не найден**. Произвольное сообщение о неудачном поиске заказа. Отображается если заказ не найден в системе или она вернула ошибку.

  Значение по умолчанию: «Извините, информация по заказу номер `{номер заказа}` не найдена».

В сообщениях можно использовать переменные:

* `{orderNum}` — цифровой номер заказа;
* `{orderDate}` — дата создания заказа;
* `{orderStatus}` — статус заказа.

### amoCRM Сохранение контакта

Сценарий передает полученные во время разговора с пользователем данные в ваш аккаунт amoCRM.

В amoCRM можно передать имя, фамилию, телефон и адрес электронной почты пользователя. Для сохранения контакта нужно заполнить как минимум одно из полей. В полях нужно указывать переменные с данными клиента. Для сохранения данных в переменных используются блоки [**Ввод текста**](text-entry.md) и [**Номер телефона**](phone-number.md).

О том как сохранять данные в переменных читайте в разделе об [обработке запросов пользователей](https://developers.sber.ru/docs/ru/salutebot/bot-steps/user-dialog/overview).

Кнопка **Связать с AmoCRM** вызывает окно авторизации в аккаунте, в котором будет сохранен контакт клиента. Данные сохраняются в разделе **Списки** → **Контакты**.

Для сохранения контакта аккаунт в amoCRM должен обладать правами администратора.

Сценарий предлагает два перехода: при успешном и неуспешном сохранении контакта.
