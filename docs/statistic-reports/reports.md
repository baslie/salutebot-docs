# Собираем метрики

Обновлено 5 сентября 2025

[![](/assets/statistic-reports/reports/salutebot-new.png)
SaluteBot](../overview.md)[![](/assets/statistic-reports/reports/Graph.png)
Graph](https://developers.sber.ru/docs/ru/va/graph/overview)

Отчеты позволяют анализировать качество работы чат-бота. Данные для отчетов можно фильтровать по времени.

Отчеты отображаются на вкладке **Статистика** для проектов, созданных в Graph или вкладке **Обзор проекта** для проектов, созданных в Code. По умолчанию первым будет открыт отчет **Диалоги**. Доступны также отчеты **Популярные интенты** и **Активность**.

Вы можете выбрать нужный отчет перейдя в соответствующую вкладку.

Отчеты **Диалоги** и **Популярные интенты** доступны только для проектов SaluteBot.

## Диалоги

Отчет содержит информацию о ключевых показателях диалогов пользователей по каждой из опубликованных версий чат-бота:

* **Всего диалогов**.
* **Без оператора** — успешные диалоги, в которых чат-бот помог пользователю не привлекая оператора.
* **Нераспознанные** — диалоги, в которых было одно или несколько срабатываний событий нераспознанной фразы (noMatch).

При установке отметки в поле **Все значения** в отчете будут содержаться все возможные показатели.

Отчет может быть представлен в виде таблицы и/или графика с ключевыми показателями, отображающими данные по дням/неделям/месяцам за выбранный промежуток в фильтре.

## Популярные интенты

Отчет содержит список интентов, которые наиболее часто срабатывали при диалоге с пользователем за выбранный период. Период по умолчанию – 1 неделя.

Таблица в отчете показывает интенты, которые используются в сценарии. Для каждого интента в списке отображаются параметры:

* **Название интента**.
* **% срабатываний в диалогах** — относительное количество попаданий в указанный интент от общего количества срабатываний интентов в сценарии.
* **Количество срабатываний в диалогах** — суммарное количество попаданий в указанный интент за выбранный период, количество считается по всем диалогам за выбранный период.

## Активность

Отчет содержит данные по количеству уникальных пользователей, переводов на оператора, сообщений и нераспознанных фраз.

Отчет состоит из блоков:

* **Пользователи**. Демонстрирует суммарное количество уникальных пользователей чат-бота за выбранный период. Здесь учитываются только уникальные пользователи, которые написали хотя бы одно сообщение за выбранный период.
* **Переводы на оператора**. Показывает суммарное количество выполненных переводов на оператора за выбранный период.
  Значение этого показателя может быть больше аналогичного в отчете **Диалоги**, так как здесь учитывается каждый факт перехода. В отчете **Диалоги** диалог, в котором было один или несколько переводов на оператора засчитывается как один диалог.
* **Сообщения**. Содержит информацию о суммарном количестве уникальных пользователей чат-бота за выбранный период. Учитываются только уникальные пользователи, которые написали хотя бы одно сообщения за выбранный период.
* **Нераспознанные**. Показывает суммарное количество нераспознанных запросов за выбранный период.

Отчет может быть представлен в виде таблицы и/или графика с ключевыми показателями, отображающими данные по дням/неделям/месяцам за выбранный промежуток в фильтре.

Если для чат-бота отсутствуют данные о диалогах в целом или для выбранного периода, будет представлено соответствующее уведомление и отчет построен не будет.

## Воронки

Отчет **Воронки** позволяет оценить эффективность разработанного чат-бота с помощью воронки конверсии и помогает отследить путь, который проходят клиенты. Вы узнаете поведение клиента на каждом шаге и поймете, как в дальнейшем улучшить сценарий.

Для сбора статистики необходимо предварительно разметить сценарий и добавить на конкретные шаги специальные метки. Таким образом, по названию меток эти шаги будут учитываться при формировании отчета.

Статистика отслеживается и собирается в соответствии с названием меток, выбранных на этапе разметки. Метки сценария – это этапы воронки для дальнейшего построения отчета.

Для каждого этапа воронки можно добавить несколько шагов сценария.

Сбор данных начинается с даты, когда на размеченных шагах сценария включен сбор статистики.

Отчет можно сформировать только по опубликованному чат-боту, где есть история переписки и переходы в размеченные экраны.

Работа с отчетом состоит из двух этапов:

1. Разметка сценария.
2. Настройка отчета.

**Разметка сценария**



Для разметки сценария:

1. Откройте сценарий чат-бота в Graph.
2. Выберите шаг сценария для создания воронки.
3. Отредактируйте название шага, указав актуальное название и цвет (опционально).
4. Перейдите в контекстное меню шага.
5. Выберите **Отслеживать статистику**.
6. Введите новое название **Метки отчета** или выберите существующую из списка.
7. Сохраните сценарий.

В результате для шага отобразится иконка отслеживания статистики и сценарий будет размечен: определены шаги, факт попадания клиента в которые сохранится для использования в отчете.



**Настройка отчета**



В настройках отчета можно выбрать формат отображения воронки.

Обратите внимание: пока в сценарии не включена опция **Отслеживание статистики**, добавить шаги в настройках **Отображаемые метки** невозможно.

Для настройки отчета:

1. Откройте проект чат-бота в Graph.
2. Перейдите в раздел **Статистика**.
3. Выберите тип отчета **Воронки**.
4. Выберите **Отображаемые метки**. В результате откроется форма для редактирования меток отчета.
5. Добавьте новую метку, если это необходимо. При добавлении выберите шаги, попадание в которые должно учитываться при отображении этого этапа воронки.
6. Включите или отключите метки для отчета, при необходимости измените последовательность меток.
7. Нажмите **Сохранить**.

В результате будет отображен отчет с учетом добавленных меток.

Для формирования отчета чат-бот обязательно должен иметь историю переписки и попадания в экраны, размеченные для отчета.

Отчет может быть представлен в виде таблицы и/или графика с ключевыми показателями, которые отображают данные по дням/неделям/месяцам за выбранный промежуток времени. Максимально доступный период – не более 3 месяцев.

**Разметка сценария**



1. Откройте сценарий чат-бота в Code.
2. Выберите стейт, попадание в который следует зафиксировать как один из этапов воронки.
3. Добавьте параметр в тэг `state` для сохранения метки `state: Node || label = Название метки`.
4. Сохраните сценарий.

В результате при выполнении сценария и попадании в выбранный стейт будет сохранен факт срабатывания, который учитывается в воронке.



**Настройка отчета**



В настройках отчета можно выбрать формат отображения воронки.

Для настройки отчета:

1. Откройте проект чат-бота в Code.
2. Перейдите в раздел **Обзор проекта**.
3. Выберите тип отчета **Воронки**.
4. Выберите **Отображаемые метки**. В результате откроется форма для редактирования меток отчета.
5. Добавьте новую метку, если это необходимо. При добавлении выберите шаги, попадание в которые должно учитываться при отображении этого этапа воронки.
6. Включите или отключите метки для отчета, при необходимости измените последовательность меток.
7. Нажмите **Сохранить**.

Отчет может быть представлен в виде таблицы и/или графика с ключевыми показателями, которые отображают данные по дням/неделям/месяцам за выбранный промежуток времени. Максимально доступный период – не более 3 месяцев.

## Фильтр пустых сессий

Для всех отчетов доступен фильтр **Скрыть пустые сессии**.

При включении этого фильтра в отчетах не будут учитываться данные по пустым сессиям и сообщениям, которые к ним относятся.

**Пустая сессия** – это сессия, в которой не было получено сообщений от пользователя. Сессия, в которой от пользователя получено только сообщение `/start`, также считается пустой.
